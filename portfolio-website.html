Building this portfolio website from scratch

HTML, CSS and JavaScript

To get a better understanding of what actually happens after the UX and UI Designers job is done - and the work is handed over to a developer in most cases - I decided to take a frontend course to learn and practice HTML, CSS and Javascript.

Challenge

The challenge was to build this responsive website to showcase my UX Design work. I wanted to enhance my frontend skills to become more confident with cooperating with developers in the future and familiarize myself with industry-standard systems.

Tools

Atom, Codepen, GitHub, Adobe XD, W3C Validation Service

Development process

I started by defining the structure and the page elements with HTML, whereafter it was time to incorporate styling and layout with a CSS stylesheet. I decided to go with a fixed header that would always be visible for the user when scrolling.

I had from the very start a mobile-first approach in my design and development process, and I used media queries to define how content should be displayed and elements styled at medium and large viewports. I also practiced doing some CSS animations and transitions. For the mobile version I wanted a hamburger menu for which I used JavaScript.

Below are the first sketches I did for the main landing page and the about me page for desktop and mobile views.

User testing

An important step in any design or development project is user testing. I wanted to test my website with 5 users to identify possible usability issues and get feedback on my portfolio site.

My test objectives were:

  1. Observe how effortlessly users are able to navigate the website - do they immediately know where to go for what information?
  2. Find out any possible usability issues.
  3. See what kind of impression the website and the copy gives of me as a Designer. Does the website showcase my skills and me as a Designer clearly?

I used four scenario tasks and four open ended questions to conduct my user studies. The usability tests didn't reveal any usability issues. All participants completed the tasks easily and quickly. However, the users had some other valuable feedback about copy and content that I acted upon.

Code quality, accessibility & cross-browser testing

The last thing I did before the launch of the website was to quality check my code and test it across browsers and devices. I used the linters W3C Validator, linter-stylelint and Standard Javascript style to check the code quality on my HTML, CSS and JavaScript files. I did some minor fixes in my code to remove identified errors or warnings. I also used atom-beautify to help format my code.

The accessibility of my UI elements such as buttons and text links I checked with the color checker and they both passed the AAA compliant.

Lastly I did a quality assurance test in the form of cross-browser testing. I found mainly some minor spacing issues that I fixed.

Final thoughts and learnings

After a steep learning curve in frontend development skills, I am ready with my website. I had some challenges on the way but with the help of my mentor and Youtube I managed to build a website that I'm happy with.

A few key learnings from this project:

  1. I'm now confident with using HTML and CSS. I also understand how JavaScript works and for what kind of functions it is used.
  2. This project helped me understand much better how to design a responsive website. Knowing the role of columns and rows in different viewports (and how to define them) was a valuable learning that will help me in my design work.
  3. I can say that I have a completely new understanding of the relationship between design and development, and I feel much more confident to talk to developers about my designs after the course.
